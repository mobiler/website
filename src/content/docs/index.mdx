## Write it once, in Rust

You write a single `MobilerApp` — typed `Model`, `Msg`, `update`, `view`. On mobile, Mobiler renders it
**natively**: SwiftUI on iOS and Jetpack Compose on Android — real native controls, no webview. The same
core also drives a **web build** (Leptos + WebAssembly) that shares most of the UI: a real browser app
from the same code, if not a pixel-for-pixel match.

## See it in one screen

```rust
fn view(&self, model: &Model) -> Widget {
    scaffold("Saldo", model.dark, tabs(model), column(vec![
        card(text(money(model, model.balance)), CardStyle::Filled),
        button("Add", ButtonStyle::Filled, Msg::StartAdd),
    ]))
}
```

That `view` runs natively on iOS and Android — and in the browser via the web build. Ready to build
something real? **[Start the guide →](/guide/)**
