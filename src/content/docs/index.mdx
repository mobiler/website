## Write it once, in Rust

You write a single `MobilerApp` — typed `Model`, `Msg`, `update`, `view` — and Mobiler renders it
**natively**: SwiftUI on iOS, Jetpack Compose on Android, and the DOM on the web. The same core,
three real native shells. No webview, no per-platform UI code.

	<Card title="One core, three platforms" icon="laptop">

## See it in one screen

```rust
fn view(&self, model: &Model) -> Widget {
    scaffold("Saldo", model.dark, tabs(model), column(vec![
        card(text(money(model, model.balance)), CardStyle::Filled),
        button("Add", ButtonStyle::Filled, Msg::StartAdd),
    ]))
}
```

That `view` runs unchanged on all three platforms. Ready to build something real?
